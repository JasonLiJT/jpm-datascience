Overview of Data Science

About this module

We will introduce you to the basics of Data Science and Machine Learning. Machine Learning is the field of Computer Science that deals with the construction of algorithms that can learn from data in order to make predictions or decisions. Machine Learning algorithms provide computers with the ability to search through data, look for patterns, and use extracted information to make educated decisions without being explicitly programmed.

Machine Learning can be applied for broadly two purposes:

  • Prediction - where the goal is to teach a machine how to accurately predict an outcome based on data that it has previously seen.

  • Inference - where the goal is to better understand and explain the data (in particular, the underlying distribution of the data).

In both cases, learning is through experience which means that if you are trying to build a predictor, its performance will depend heavily on how similar the data you used to train the predictor is to the problem you are applying it to. For example, if you build a predictor based on climate data from Hawaii, you shouldn’t expect it to perform well when predicting the behaviour of the London stock market (unless you know something you shouldn’t…​)!

Examples of Machine Learning problems in the real world

Recommender systems

Netflix is a movie rental company in the US. In 2006, the company announced the Netflix Prize, a Machine Learning and Data Mining competition for movie rating prediction. Netflix in based on a movie recommender system that changes according to the user’s personal movie rentals. If a user frequently gives high ratings to comedy movies or TV shows, Netflix recommends other comedy-related videos. So, one of the goals of recommender systems is to try and predict what the customers would think of the other movies based on what they rated so far.

Supervised and unsupervised learning

We can distinguish between two main types of Machine Learning techniques: supervised learning techniques and unsupervised learning techniques:

  • Supervised learning: the data is presented to the algorithm with example inputs and their associated outputs. This can be either a classification or a regression problem.

    • Classification: outputs are discrete classes and we want to learn a general rule from the input-output mappings of the labeled data that can be used to predict the classes of unlabeled data (i.e. data that are completely new and unseen).

    • Regression: outputs are continuous variables as opposed to discrete classes.

  • Unsupervised learning: no labels are given to the learning algorithm. The goal is to discover groups in the data (clustering) or to determine the distribution of data within the input space (density estimation). Techniques for dimensionality reduction, such as feature extraction, can also be considered to be unsupervised learning algorithms.

Figure 1. Supervised vs. Unsupervised Learning.

In classification, a model is trained to recognise the relationship between a particular set of data and a class (a single discrete value), based on data that is known to belong to that class (which is presented as the training data). In other words, the training data provides the algorithm with “experience” of which inputs (usually a vector of discrete and/or continuous predictor values) are associated with classes. The “experience” is in the form of individual observations, which in the datasets we will be working with are represented by the rows. When the trained classifier is presented with a set of new inputs, it then outputs the class based on this “experience”.

  • The output variable goes by various names: yy, dependent variable, response or target.

  • The predictor measurements also go by various names: XX, inputs, regressors, covariates, features, or independent variables.

It is important to note that XX is in upper case to indicate that it is usually a matrix of values with each column corresponding to a single feature and the values taken for that feature by all of the instances/observations. yy is in lower case because it represents a vector, with only a single value associated with each observation.
