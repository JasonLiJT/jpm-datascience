Lexicon

Here are some quick definitions to help you review the concepts in this workshop:

Accuracy

The rate of correctly classified predictions (samples) made by the model over a particular input dataset.

Attribute (variable, feature)

A characteristic describing an instance. In matrix terms, features are represented by the columns. The type of the attribute is usually either (a) categorical (taking a finite number of discrete values) or (b) continuous (taking quantitative values)

Bagging (bootstrap aggregating)

Bootstrap aggregating (bagging) aims to reduce variance and helps to avoid overfitting. Bagging takes repeated samples with replacement from the single training set in order to generate b different bootstrapped training data sets. Each of these data sets is used to train a different model. The final output is calculated by uniformly averaging the individual predictions or applying majority voting over class labels.

Bias

The bias is error from erroneous assumptions in the learning algorithm. High bias can cause an algorithm to miss the relevant relations between features and target outputs (underfitting).

Boosting

Boosting involves incrementally building an ensemble by training models sequentially. A succession of models are built iteratively, with each one being trained on a data set in which points misclassified by the previous model are given increased weight. Finally, all of the successive models are weighted according to their success and then the outputs are combined using voting or averaging. By far, the most common implementation of boosting is the Adaboost algorithm.

Bootstrapping

A validation technique that relies on the training records being randomly sampled with replacement multiple times. If the original data matrix consist of N samples, it can be shown that, on average, a bootstrap sample of size N contains approximately 63.2% of the samples from the original data (.632 bootstrap). The remaining samples that are not included in the bootstrap training set become part of the bootstrap test set. The sampling process is then repeated many times to generate b bootstrap samples of the same size.

Bootstrap aggregating

See bagging

Class (label, target, response)

A given discrete value from a small, mutually exclusive set of labels used as target variables in a classification problem.

Classification

The family of supervised learning problems where outputs are discrete classes.

Classifier (classification model)

A function that best attempts to capture and map the relationship in-between unlabelled instances (input data) to discrete classes (output classes).

Clustering

The field of unsupervised learning that attempts to discover the intrinsic grouping and patterns in a set of unlabelled data by identifying similarities between objects based on their characteristics.

Confusion matrix (contingency table)

A specific table layout that allows visualisation of the performance of a classification model. Each column of the matrix represents the instances in a predicted class while each row represents the instances in an actual class (or vice-versa). The size of the matrix will be equal to m×mm×m , where mm is the number of distinct classes. The confusion matrix can be used towards calculating performance metrics such as the overall accuracy, sensitivity and specificity, among others.

Cross-validation (k-fold cross-validation)

In k-fold cross-validation, the entire dataset is randomly split into k mutually exclusive subsets (folds) of approximately equal size. The algorithm of k-fold cross-validation performs k iterations in total. In each iteration, one subset is considered to be the test set, while the remaining folds are used to train the classification model.

Data pre-processing (data pre-treatment)

Real-world data are generally incomplete, noisy and inconsistent. In order to avoid "garbage-in, garbage-out" through the analysis process, the application of data pre-processing techniques is crucial. Data pre-processing can make the difference between a useful model and no model at all. During this step, input data are most commonly scaled, centered and/or transformed.

Decision Trees

Decision Tree classifiers construct classification models in the form of a tree structure. A decision tree progressively splits the training set into smaller subsets. Each node of the tree represents a subset of the data.

Ensemble models

Ensemble learning involves the combination of several models towards solving a single prediction problem. Ensemble models aim at improving the stability and performance of individual classifiers by generating multiple models which learn and make predictions independently. Those predictions are then combined into a single (mega) prediction that should be better, on average, than the prediction made by any one classifier. Empirically, ensembles tend to yield better results when there is significant diversity among the models. There are two well-known categories of ensemble learning models (a) bootstrap aggregating (“bagging”) and (b) boosting

Error rate

The rate of misclassified predictions (samples) made by the model over a particular input dataset.

Exploratory Data Analysis (EDA)

Exploratory data analysis is the field dealing with the analysis of data sets as a means of summarising their main characteristics, most often using visual methods.

F1

F1 measures the accuracy of the model with respect to a particular class, and is the harmonic mean of precision and recall. It is defined by: F1=2∗(precision∗recall)/(precision+recall)F1=2∗(precision∗recall)/(precision+recall)

Feature

See attribute

Generalisation

Generalisation is the ability of a Machine Learning model to perform accurately on new, unseen data after being built and trained on a training data set.

Holdout

A validation technique where the input dataset is split into two mutually exclusive datasets, the train and test set (usually, 70% of the input data end up in the training set and 30% of the data end up in the test set).

Instance (example, sample, case, record)

A single object (in matrix terms, it would represent a single row) of the input data which is used to train a model. In most Machine Learning applications, instances are described by a set of features.

k-fold cross-validation

See cross-validation

kNN

The k Nearest Neighbours algorithm is a non-parametric, instance-based (lazy) learning method used for classification and regression. The principle behind kNN is very simple. Given a dataset where you know the class labels (y values), when a new point is introduced, you want to find a particular number of points in the data closest in distance to the new point. These are called the “nearest neighbours”. You then use the labels associated with these nearest neighbour points (which may or may not be different from each other) to predict the label of the new point.

Label

See class

Machine Learning

"The field of study that gives computers the ability to learn without being explicitly programmed" (Arthur Samuel, 1959).

Overfitting

The situation where a trained model may perform extremely well on the training dataset, but can still fail to predict new unseen data, leading to poor generalisation performance.

Precision

Precision is a measure of the accuracy for a specific class, it reports the proportion of correct classifications for a specific class. It is defined by Precision=tp/(tp+fp)Precision=tp/(tp+fp) where tptp and fpfp are the numbers of true positive and false positive predictions for the considered class, e.g. ability to correctly classify wine as being of high quality or poor quality. tp+fptp+fp is the total number of elements labelled as belonging to the considered class.

Random Forests

Random forests belong to a class of machine learning methods called ensemble learning, where many models are generated from the training data, and they are used together to make predictions. In the case of random forests, the individual models are decision trees.

Regression

The family of supervised learning problems where outputs are continuous as opposed to discrete classes.

Recall

See sensitivity

Sensitivity (Recall or True Positive Rate)

Sensitivity reports the ability of a model to select instances of a certain class from a dataset. It is defined by Recall=Sensitivity=tp/(tp+fn)Recall=Sensitivity=tp/(tp+fn) where tptp and fnfn are the numbers of true positive and false negative predictions for the considered class. tp+fntp+fn is the total number of elements that actually belong to the considered class.

Specificity (True Negative Rate)

Specificity reports the ability of the model to correctly exclude class non-members in a dataset from the class. It is defined by Specificity=tn/(fp+tn)Specificity=tn/(fp+tn) where tntn and fpfp are the numbers of true negative and false positive predictions for the considered class. fp+tnfp+tn is the total number of elements that should not be included in the class.

Supervised Learning

The type of Machine Learning model where the data are presented to the algorithm with example inputs and their associated outputs. This can be either a ‘classification’ or a ‘regression’ problem.

Target

See class

True Positive Rate

See sensitivity

True Negative Rate

See specificity

Underfitting

The situation where a model might be too simplistic, unable to capture the true relationship in the data that we observe. These models would be said to have high bias; they do not fit the data very well, which leads to a high generalisation error on new test data.

Unsupervised Learning

The type of Machine Learning model where no labels are given to the learning algorithm. The goal is to discover patterns in the data (clustering) or to determine the distribution of data within the input space (density estimation). Techniques for dimensionality reduction, such as feature extraction, can also be considered to be unsupervised learning algorithms.

Variance

The variance is error from sensitivity to small fluctuations in the training set. High variance can cause overfitting: modelling the random noise in the training data, rather than the intended outputs.
